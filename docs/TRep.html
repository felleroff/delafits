TRep

Types ››

Unit: DeLaFitsCommon


Type: type


TRep = (repUnknown, rep80f, rep64f, rep32f, rep08c, rep08u, rep16c, rep16u,

   rep32c, rep32u, rep64c);


Description:


TRep defines the user data presentation: integers or numbers from a floating comma (various bit). This type of buffer to read / write data, see. TFitsFrame. The Standard defines 9 types (including sign extension) of data format. Delphi and Lazarus define more numerical types, for example Word, Integer, Double, etc. DeLaFits supports a 10 of them.


  • repUnknown - unknown type of the buffer of the user;


  • rep80f - Extended;


  • rep64f - Double;


  • rep32f - Single;


  • rep08c - ShortInt;


  • rep08u - Byte;


  • rep16c - SmallInt;


  • rep16u - Word;


  • rep32c - LongInt;


  • rep32u - LongWord;


  • rep64c - Int64;


Note:


DeLaFits determines the best option of data representation automatically (analyzes BITPIX, BZERO and BSCALE). But, the user can specify the type of buffer read/write, for example:


Fits.DataRep := rep32c;


It means that the read / write of data will be carried out with a buffer type:


Buffer: array of array of LongInt;


Thus, TFitsFrame defines Buffer as a two-dimensional array of integral numbers LongInt:


Fits.DataPrepare(Pointer(Buffer), Rgn);

Fits.DataRead(Pointer(Buffer), Rgn);

Fits.DataWrite(Pointer(Buffer), Rgn);
