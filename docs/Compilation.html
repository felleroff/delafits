Compilation

At compilation (build) your application to improve performance and economy of resources in DeLaFits, you can use the extra settings for the compiler (conditional defines). Be careful, some directives can not be used in multi-threaded applications.


Buffer of read/write


The library uses an intermediate buffer for reading/writing data. You can define methods for creating and release of memory for this buffer.


  • DELADATBUFSHARED − set a common, shared buffer. All objects use the a single buffer which is initialized once. This directive will increase the performance of successive operations with a series of files and will reduce the fragmentation of memory. But this mode leads to a well-known problems in a multi-threaded application. In this case it is necessary to synchronize the methods of read and write.


  • DELADATBUFPRIVATE − each object works with its buffer. The object initializes its buffer once. This mode allows you to use an object in a multi-threaded application and not worry about syncing read and write operations. But this mode boosts consumption and memory fragmentation in proportion to the number of instances of the class.


  • DELADATBUFTEMP − each object creates a local buffer, as required, and immediately deletes it after use. This mode saves memory, thread safe, but will fragment a memory and slows the intensive operations of read and write. Use a this mode when the number of objects in the application is limited, or operations of read/write are not performed often.


The default mode is DELADATBUFSHARED


Support BitPix


The standard defines the format of data storage: -64, -32, 8, 16, 32 and 64. This is the integers 8, 16, 32 bits or real numbers with a single or double precision. The format is set by keyword BITPIX in file header. You can enable in your application, only certain values BITPIX. This will reduce the size of DeLaFits into your application.


  • DELABITPIX64F - BitPix = -64;


  • DELABITPIX32F - BitPix = -32;


  • DELABITPIX08U - BitPix = 8;


  • DELABITPIX16C - BitPix = 16;


  • DELABITPIX32C - BitPix = 32;


  • DELABITPIX64C - BitPix = 64;


Default DeLaFits supports all formats − all conditional directives of DELABITPIX are included.


Conditional directives are defined in DeLaFitsCompiler.inc
