Description:


The constants define the rules of parse/compose of the line of header: TLineItem.Value in TLineBuilder.


Line is a container (record) of the form:


<keyword-separator-value-separator-note>


  • keyword - keyword name;
  • separator - indicator that separates the keyword and the value, symbol '=';
  • value - value of a line;
  • separator - indicator that separates the value and the note, symbol '/';
  • note - comment to the value.


List of const:


  • cCast - Basic constant, not used;


  • cCastChars - The value of the line is analysed "as is", length of lines = 80 characters, see Example;


  • cCastCharsLong - The value of the line is analysed "as is", length of lines ~80 characters. The keyword will be duplicated, see Example;


  • cCastString - The value of the line is analysed as quote: the value in quotes, the quotes inside the value is duplicated, see Example;


  • cCastBoolean - The value of the line is analysed as Boolean, see Example;


  • cCastInteger - The value of the line is analysed as Integer (Int64), see Example;


  • cCastFloat - The value of the line is analysed as Float (Extended), see Example;


  • cCastRa - The value of the line is analysed as Float (Extended) - Right ascension α, see Example;


  • cCastDe - The value of the line is analysed as Float (Extended) - Declination δ, see Example;


  • cCastDateTime - The value of the line is analysed as TDateTime, see Example;


  • cCastDate - The value of the line is analysed as TDateTime (Date only), see Example;


  • cCastTime - The value of the line is analysed as TDateTime (Time only), see Example;


To read and rewrite the value of the line of the header use the properties of TLineBuilder: ValuesChars, ValuesCharsLong, ValuesInteger etc.
