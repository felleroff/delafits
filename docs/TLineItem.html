TLineItem

Types ››

Unit: DeLaFitsCommon


Type: type


TLineItem = record

 Keyword: string;

 ValueIndicate: Boolean;

 Value: record

   Str: string;

   Bol: Boolean;

   Int: Int64;

   Ext: Extended;

   Dtm: TDateTime;

 end;

 NoteIndicate: Boolean;

 Note: string;

end;


Description:


The standard describes the header of file as a list of lines. The length of line - 80 characters. Each 80-character header keyword record shall consist of a keyword name, a value indicator (only required if a value is present), an optional value, and an optional comment. TLineItem is a container (record) of elements of such line:


<keyword-separator-value-separator-note>.


  • keyword - keyword name;
  • separator - indicator that separates the keyword and the value, symbol '=';
  • value - value of a line;
  • separator - indicator that separates the value and the note, symbol '/';
  • note - comment to the value.


Keyword: string;


Keyword name;


ValueIndicate: Boolean;


The Value Indicator. Separator of the keyword and the value. 'True' - there is a separator, 'False' - no separator;


Value: record

 Str: string;

 Bol: Boolean;

 Int: Int64;

 Ext: Extended;

 Dtm: TDateTime;

end;


The value of a line. The value can have various type: string, Boolean, TDateTime etc. In DeLaFits is defined this type 'Value: record...' (analogue Variant) - universal container. cCast constant defines the field TLineItem.Value, which contains the real data;


NoteIndicate: Boolean;


Separator of the value and the note. 'True' - there is a separator, 'False' - no separator;


Note: string;


Comment to the value.
